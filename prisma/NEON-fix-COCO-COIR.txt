If `prisma migrate deploy` fails with P1002 (advisory lock timeout):
  Neon’s *pooler* host (`…-pooler…`) often cannot acquire Postgres advisory locks. Use a *direct* connection for migrate only.

  In Neon: Dashboard → your project → Connection details → copy the **Direct** connection string (host has NO `-pooler` in the name).

  From your machine (use Direct URL only for this command; keep pooler URL in Vercel for the app):

     cd /path/to/web
     DATABASE_URL="postgresql://USER:PASS@ep-xxxx.region.aws.neon.tech/neondb?sslmode=require" npx prisma migrate deploy

  (Replace with your Direct string from Neon; same user/password as pooled, hostname without -pooler.)

  prisma.config.ts preserves DATABASE_URL from the shell - it will NOT be overwritten by .env / .env.local anymore.

  Or paste and run in Neon **SQL Editor** (no Prisma needed):
     prisma/migrations/20260413210000_diary_medium_coco_peat/migration.sql

---

Otherwise do ONE of:

  A) From your machine - Direct DATABASE_URL as above, then:
     npx prisma migrate deploy

  B) Neon Dashboard → SQL Editor
     Paste and run the full contents of:
       prisma/migrations/20260413210000_diary_medium_coco_peat/migration.sql

After that, redeploy is not required; the next request should work.
